## Implementation Context

Plan 01 creates:
- `functions/index.js` - Edge function entry point
- `lib/weather.js` - OpenWeatherMap client
- `lib/cache.js` - KV storage with TTL
- `lib/i18n.js` - Chinese weather translations

These files expect environment variables:
- `OPENWEATHER_API_KEY` - Required, from openweathermap.org
- `CITY` - Optional, defaults to "Beijing"
- `WEATHER_KV` - KV namespace binding (set in EdgeOne console)

## Deployment Requirements

Per DEP-01: EdgeOne Pages deployment
Per DEP-02: Document API key setup
Per DEP-03: Document Feishu signature URL setup

## Feishu Signature Integration

Feishu (Lark) fetches the page and extracts the `<title>` tag content for the signature.
The deployed URL should be entered in Feishu settings as the signature URL.
